Данный код выведет в консоль: "Bad: undefined".
Такой результат получается из-за использования объявления переменной с помощью var, которая имеет функциональную область видимости. В связи с этим цикл завершает свою работу быстрее, чем успевает пройти 3 секунды, заданные в методе setTimeout. 
(1)Если заменить var на let (блочная зона видимости), то код выдаст нужный результат.
(2)Также можно добавить замыкание с помощью обёртки в виде функции поверх метода setTimeout. Это позволяет сохранить текущее значение переменной i для каждой итерации цикла, обеспечивая корректное выполнение метода setTimeout с доступом к правильному индексу массива arr[i].
